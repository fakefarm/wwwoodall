use the middleman

Jan 01, 2015

The exercise of formulating ideas for the general public is a great way to solidify thought. Substantial content typically requires concentration which is not an attribute of blogging platforms. Their dashboards contain so many mysterious buttons, links to other blogs, and advertisements to ignore. All of which combine for a massive A.D.D. hive.

One Solution is to write in a plain text editor, like Sublime Text, then copy/paste your final version.

Alternatively, if you choose to publish directly from the platform, then your content is stored there. If they shut down before you get the content, it's gone. (Read, Posterous).

Content control should begin and end with the writer instead of the other way around.

A blog about my blog

As a developer, I am learning things constantly. And I think with pen and paper so I have stacks of notes. I am okay with this analog approach but my notes have reached a level of quantity that I want to be able to reference these things.

Problem with a blog

Blogs are not reference like. I want something I can quickly jot a snippet and then reference it later.

Wiki & other Resource platforms

I’ve looked into github’s wiki and sites like pineapple.io. While I appreciate them, I don’t want to build my asset library on top of a third party.

Building and Publishing

I think that I can admit that I want to build as well as publish.

I’m hooked on middleman and I want to learn how to use it.

This is sort of a meta post in that I felt like I’m going to write it in good faith that one day I’ll have a place and story which this fits in.

So, this is my first post into becoming an expert in the ARM stack. The ARM stack is Angular, Rails, and Middleman.

So, in order to become an expert in that stack, I need to know some intermediate concepts with middleman.

The 2nd blog

So, in my quest to start documenting my stuff, I wanted to build a second blog, which middleman allows.

I’ve hit a few problems like; - middleman article ‘new article’ no longer works - all my blog posts have disappeared from my index. (The content is still there, just the index is gone.) -

My goals

  • to begin to write and record my progress on a regular basis
  • build a wiki style reference system
  • to take a deep dive into middleman

Why middleman

Speed

The biggest reason to go with middleman is speed. Static sites are not bound to cache or server loading of data. Data is already there.

Rails for static sites

As a rails developer first, I love having a similar experience that I can work with Ruby and the Gems I love, like the ThoughtBot Suite.

It’s Simple

Once you spend a little time on this you will see that it’s really easy to get going

Asset pipeline

I get to develop with slim, coffee, and scss for static sites.

How I will host my angular app

I would rather work here than fire up a node and gulp and learn all those other build tools. I’m sticking with my favorites for now.

Middleman gists

meta data

current_page current_article helper-methods (in config) front matter

Access frontmatter;

metadata[:page]['action']

params

Middleman params are what’s found in a url for things like;

/notes?action=todo&asset=image To access them, you use the

req.params @params={"action"=>"todo", "asset"=>"image"}>

if you use req.params['action']

you’ll get back ‘action’

article.url

for some reason article.url adds a trailing ‘/‘ that prevents my page template from noticing it. SO, I had to add articl.url.slice(1..-2) and it works. Bug?

Middleman Data

Data is great because you can create topics of yml files that will be categories of content. I have started some as little places to store information that can be public.

It’s a yaml file and I can add content and the template will iterate through them.

middleman has

  1. images/assets
  2. data.yml
  3. config.rb
  4. template.rb
  5. styles
  6. overrides
