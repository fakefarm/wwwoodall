When $(document).ready() is not ready

May 27, 2015

I wrote this javascript and it worked.

    $(document).ready(function(){
       // all
       // my
       // awesome
       // code 
    });

Then, I had to implement the same behavior on a different page but this time in a modal and it didn't work. So I showed it to Shaun (@shaunrussell shwing) and he immediately spotted the problem. He explained that markup that comes through via AJAX is not in the DOM when the page loads and that I should move my code into a function that I can explicitly fire.

    function runMyCode() {
       // all
       // my
       // awesome
       // code 
    }

   document.ready(function(){
        runMyCode()
    });

then, for the modal, I add an onClick handler

    $("#my-modal").click(function(){
        runMyCode()
    });

It makes sense now, but the lesson learned is that the DOM changed after $(document).ready() so there was nothing for me to attach to.

Shaun also explained another way to approach it. Which is to watch the DOM on an element you know will be there. 

$("#some-parent-already-in-the-DOM").on('click', "#my-modal", function(){
    runMyCode()
});

This way, anytime #some-parent changes, it will look to see if #my-modal has changed. It's not as performant but is a bit more dynamic.
