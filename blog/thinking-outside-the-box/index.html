Thinking outside the box

Dec 02, 2014

There's a clothing brand called RVCA. I used to think they were cooler than I was until they sold out to Billabong. Now they're just part of the machine so they're not cool anymore. (Well, maybe they are. I dunno)

At Starter League I learned to build apps with a strict regimine of Routes, Controller, Action, View. When all of this stuff was new I was trying to come up with a way to remember the sequence. I remembered it by saying to myself

It's like RVCA, but better :-)

That RCAV pattern was so tightly compacted that I didn't recognize that it can be broken apart to do many other cool things.

One of the patterns we use at LivingSocial is to leave off the "V" (View) and render json, or html in a different location.

That's handled through AJAX which can call the route and track the controller/action and get the necessary data. As I explain this concept It seems basic and I even question why I would write it. But the value here is one that's universal - learning something in a strict way can be challenging to think of it in non-strict ways. It's effectively learning to think outside the box. That was the hurdle I overcame in seeing RCAV differently.

So instead of RCAV, with Ajax we can call it RCAA

  1. Setup the familiar Route, Controller, and Action.
  2. Make an ajax call to that route
  3. That action will the provide the data requested and describe the render option - json, html, or text.
  4. Since AJAX is calling it, then the new data will be part of an existing view rather than a new one.

That's all for now, but maybe I'll add some code snippets later..
