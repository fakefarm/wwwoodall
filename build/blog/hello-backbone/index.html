hello backbone

May 29, 2015

I recently started on a new team at work that uses BackboneJs. I was a bit nervous having already invested so much time learning Angular but I've been pleased with what I've found in my preliminary research.

What I've since learned is that BackboneJs was abstracted off a Rails app. This has made things somewhat intuitive. for example, BackboneJs uses models and they seem similar to how Rails uses models. There are initializers, validations, and everything is managed from a js object syntax.

I'm still a rather new developer so I can't imagine I'm going to encounter the really sophisticated problems that Angular wants to attack. I hear BackboneJs requires more setup than Angular but that also means less magic.

I already finding where my learning with Angular seemed to need to read every line of how things work, I'm finding that I can skim the books and reference them when I discover something I want to do.
