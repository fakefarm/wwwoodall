Use different layouts in Rails

Jul 08, 2015

hashpage will be offering different page themes based on a tweet. For example a tweet might look like;

List Light Newspaper #theme

— dave woodall (@wwwoodall) July 8, 2015

At first, I imagined having a single application and similar DOM that would all be managed by simply styles. Like what Zen Garden has shown possible with simple CSS. However, I started to realize that Not only will the theme look different, I want tweets to display different tweets, differently.

So I reached out to @iamlacroix who helped me figure out a different approach. We decided that each theme should have it's own layout and not try to fit themes into standard applciation.html.erb.

We came up with a solid approach of building a ThemeService which will select which the theme if real or use the default in a case like typo.

class ThemeService
  THEMES = %w(beginner advanced another more)
  DEFAULT = 'beginner'

  def self.set_layout(theme)
    layout_dir = THEMES.include?(theme) ? theme : DEFAULT
    layout_dir + '/base'
  end
end

Then, the layout would be set in a before_action in the controller with something like

before_action :get_theme

def get_theme
    @theme = ThemeService.set_layout(theme_tweet)
end

I'm still working on this but my current assumption is that each index will then render the theme's layout with something like;

def index
  render layout: @theme
end

Layout file structure

I'm planning to have my layouts in this order unless it doesn't work..

layouts/
  default/
    _app.html.erb
    _nav.html.erb

Stylesheets structure

Then, accompanying stylesheets with something like. In the past I've managed all screens in the same file but I want to try to give them each their own space without clutting other screens. We'll see how it goes.

stylesheets/
  themes/
    default/
      nav/
        _desktop.scss
        _mobile.scss
        _tablet.scss
      footer/
        _desktop.scss
        _mobile.scss
        _tablet.scss
      _base.scss
